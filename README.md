# Proyecto El Mirador - Gestión de Gastos Comunes

## Descripción del proyecto

Este proyecto corresponde al desarrollo de un prototipo funcional para el edificio **El Mirador**, orientado a mejorar la administración de gastos comunes, pagos, morosidad, residentes, solicitudes e informes.

La problemática principal del caso es que la administración necesita contar con una herramienta más clara, ordenada y eficiente para controlar la información de los residentes, registrar pagos, identificar deudas y gestionar solicitudes de la comunidad.

El prototipo busca entregar una solución visual y funcional que facilite la gestión administrativa del edificio y mejore la transparencia de la información.

---

## Objetivo del sistema

Diseñar un prototipo funcional que permita a la administración del edificio **El Mirador** gestionar de forma centralizada:

* Residentes del edificio.
* Pagos de gastos comunes.
* Morosidad.
* Recordatorios de pago.
* Solicitudes y quejas.
* Informes administrativos.

---

## Módulos del prototipo

El prototipo incluye los siguientes módulos principales:

### 1. Inicio de sesión

Permite el acceso al sistema mediante correo y contraseña, simulando un ingreso seguro para la administración del edificio.

### 2. Panel principal

Muestra un resumen general de la gestión del edificio, incluyendo información de pagos, morosidad, residentes y solicitudes.

### 3. Gestión de residentes

Permite visualizar residentes registrados, revisar sus datos principales y acceder a la opción de agregar nuevos residentes.

### 4. Gestión de pagos

Permite revisar el estado de los pagos de gastos comunes, identificando pagos realizados, pendientes y morosos.

### 5. Morosidad

Permite visualizar residentes con deuda, montos pendientes y opciones para enviar recordatorios de pago.

### 6. Solicitudes y quejas

Permite registrar y revisar solicitudes realizadas por los residentes, indicando su estado de atención.

### 7. Informes

Permite visualizar información resumida para apoyar la toma de decisiones administrativas.

## Modelo de calidad aplicado

Para evaluar el prototipo se consideraron criterios asociados al modelo de calidad **ISO 25000**, principalmente:

### Adecuación funcional

El sistema responde a las necesidades principales del caso, permitiendo gestionar residentes, pagos, morosidad, solicitudes e informes.

### Usabilidad

La interfaz fue diseñada para ser clara, ordenada y fácil de entender, permitiendo una navegación simple para el usuario administrador.

### Seguridad

El prototipo considera una pantalla de inicio de sesión como punto de acceso al sistema.

### Mantenibilidad

La estructura del prototipo está separada por módulos, lo que facilita futuras mejoras o modificaciones.

### Fiabilidad

El sistema permite visualizar información relevante para apoyar la gestión administrativa y reducir errores en el seguimiento de pagos y morosidad.
